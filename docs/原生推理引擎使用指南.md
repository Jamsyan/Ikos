# 原生推理引擎使用指南

## 概述

原生推理引擎是 Ikos 系统的核心组件，基于 Transformers + PyTorch 实现，支持从 4GB 显存到 24GB+ 显存的多种硬件场景。

## 核心特性

- **硬件自动检测**：自动检测 GPU、CPU、内存并分级
- **显存精细化管理**：VRAM 池化、优先级调度、抢占机制
- **多量化等级支持**：NF4/INT4/INT8/FP16/FP32
- **CPU-GPU 混合推理**：突破显存限制，4GB 显存可运行 7B 模型
- **智能引擎切换**：根据硬件自动选择最优引擎模式

## 快速开始

### 1. 安装依赖

```bash
# 使用 uv 安装所有依赖
uv add torch transformers accelerate bitsandbytes pynvml psutil
```

### 2. 基础使用

```python
from ikos.core import create_native_engine

# 创建引擎（自动检测硬件和推荐量化）
engine = create_native_engine(quantization="auto")

# 调用模型
response = engine.call(
    prompt="请介绍一下傅里叶变换",
    model="Qwen/Qwen2.5-7B-Instruct",
    max_tokens=1024
)

print(response.content)
```

### 3. 手动指定量化等级

```python
# 根据硬件选择量化等级
# 4GB 显存：使用 NF4
# 8GB 显存：使用 INT8
# 16GB 显存：使用 FP16
engine = create_native_engine(quantization="NF4")
```

## 硬件分级与推荐

| 场景     | 显存   | 推荐量化 | 可运行模型         |
|----------|--------|----------|------------------|
| 极限场景  | 4GB    | NF4      | 7B (CPU-GPU 混合) |
| 基础场景  | 8GB    | INT8     | 7B (GPU)         |
| 性能场景  | 16GB   | FP16     | 14B (GPU)        |
| 旗舰场景  | 24GB+  | FP32     | 35B+ (GPU)       |

```python
from ikos.core import detect_hardware

hardware = detect_hardware()
print(hardware)

```

### 显存管理

```python
from ikos.core import VRAMManager, Priority

# 创建显存管理器
vram = VRAMManager.from_hardware(hardware)

# 分配显存（高优先级）
vram.allocate(4.0, "qwen-7b", Priority.P1)

# 查看使用情况
usage = vram.get_usage()
print(usage)

# 释放显存
vram.release("qwen-7b")
```

### 量化推荐

```python
from ikos.core import auto_recommend_quantization

# 为特定模型推荐量化等级
config = auto_recommend_quantization(
    model_name="qwen3.5:7b",
    hardware_info=hardware
)

print(f"推荐量化：{config.level.value}")
print(f"显存占用：{config.calculate_memory_usage(7.0):.1f}GB")
```

### 引擎切换

```python
from ikos.core import EngineSwitcher, EngineType

# 创建切换器
switcher = EngineSwitcher()

# 自动选择引擎
engine_config = switcher.select_engine()

# 手动切换到原生引擎
engine = switcher.switch_to(EngineType.NATIVE)

# 查看引擎信息
info = switcher.get_engine_info()
print(info)
```

## 模型下载

原生引擎会自动从魔塔社区下载模型：

```python
from ikos.core import NativeModelLoader

loader = NativeModelLoader(cache_dir="./data/models")

# 下载并加载模型
model, tokenizer = loader.load_model(
    model_name="Qwen/Qwen2.5-7B-Instruct",
    quantization="NF4"
)
```

### 支持的模型源

- **魔塔社区**（优先）：国内访问快
- **Hugging Face**：备选

### 模型缓存

模型会自动缓存到 `./data/models` 目录，重复使用不会重新下载。

## 多模型投票

```python
from ikos.core import create_native_engine

engine = create_native_engine()

# 多模型投票
result = engine.vote(
    prompt="请解释量子力学",
    models=[
        "Qwen/Qwen2.5-7B-Instruct",
        "Qwen/Qwen2.5-3B-Instruct",
        "deepseek-ai/DeepSeek-R1-Distill-Qwen-7B"
    ],
    voting_strategy="majority"
)

print(f"获胜模型：{result.winner_model}")
print(f"最终答案：{result.winner_content}")
```

## 性能优化建议

### 1. 选择合适的量化等级

- **速度优先**：INT8（质量损失小，速度快）
- **显存优先**：NF4（显存占用最低）
- **质量优先**：FP16/FP32（显存充足时）

### 2. 批量推理

```python
# 批量调用多个模型
responses = engine.call_batch(
    prompt="相同的问题",
    models=["model1", "model2", "model3"]
)
```

### 3. 模型卸载

```python
# 卸载不用的模型释放显存
engine.unload_model("model_name")

# 或卸载所有模型
engine.unload_all_models()
```

## 常见问题

### Q1: bitsandbytes 安装失败

**问题**：`pip install bitsandbytes` 报错

**解决**：
```bash
# Windows 用户
pip install bitsandbytes-windows

# 或降级使用 INT8
engine = create_native_engine(quantization="INT8")
```

### Q2: 显存不足 OOM

**问题**：运行时提示 CUDA out of memory

**解决**：
1. 降低量化等级（FP16 -> INT8 -> NF4）
2. 使用更小的模型（7B -> 3B）
3. 启用 CPU-GPU 混合推理（自动）

### Q3: 模型下载慢

**问题**：从魔塔社区下载速度慢

**解决**：
```python
# 使用镜像源
export MODELSCOPE_CACHE="./data/models"
```

### Q4: 推理速度慢

**问题**：生成速度只有几个 token/s

**解决**：
- 这是正常现象，尤其是 NF4 量化
- 使用 INT8 量化可提升速度
- 批量推理可提高效率
