系统架构
========

Ikos 采用四阶段流水线架构，每个阶段都有明确职责和输出。

总体架构
--------

.. mermaid::

   flowchart TB
       INPUT["用户输入"] --> Stage1["第一阶段：需求解析"]
       Stage1 --> Stage2["第二阶段：智能检索"]
       Stage2 --> Stage3["第三阶段：数据筛选"]
       Stage3 --> Stage4["第四阶段：输出分流"]
       Stage4 --> OUTPUT["知识图谱 + 结构化数据"]

四阶段详解
----------

第一阶段：需求解析
~~~~~~~~~~~~~~~~~~

**职责**：将用户自然语言转换为工程化提示词

**核心机制**：
- 多轮转换
- 旁系监督
- 网络验证
- 球形知识空间构建

**输出**：工程化提示词、核心圈定义

第二阶段：智能检索
~~~~~~~~~~~~~~~~~~

**职责**：并行搜索获取海量原始数据

**核心机制**：
- 多模型拆分搜索
- 多品审核
- 备忘录迭代评审

**输出**：原始 HTML 数据、备忘录评审结果

第三阶段：数据筛选
~~~~~~~~~~~~~~~~~~

**职责**：筛选数据并构建知识图谱

**核心机制**：
- 分合策略
- 多模型投票
- 知识图谱同步构建
- 模型重写

**输出**：结构化数据、知识图谱

第四阶段：输出分流
~~~~~~~~~~~~~~~~~~

**职责**：根据用户配置输出结果

**核心机制**：
- 用户配置读取
- 模板输出
- 数据库输出

**输出**：Markdown/PDF文档、JSON 文件、数据库记录

核心组件
--------

ModelProvider
~~~~~~~~~~~~~

多模型统一调用接口，支持：
- Ollama 本地模型
- OpenAI 兼容 API
- 多模型并行调用
- 投票决策

SearchProvider
~~~~~~~~~~~~~~

搜索引擎抽象接口，支持：
- Playwright 浏览器自动化
- 多搜索引擎适配
- 网页内容抓取

VoteEngine
~~~~~~~~~~

多模型投票引擎，实现：
- 多数投票策略
- 加权投票策略
- 结果汇总与决策

技术栈
------

- **模型层**: Ollama + Qwen 3.5 / DeepSeek-R1
- **搜索层**: Playwright（浏览器自动化）
- **数据层**: Chroma/Milvus + Neo4j
- **UI 层**: PyQt6
- **配置层**: YAML

设计原则
--------

1. **不依赖模型记忆**：使用三方信息求证
2. **多模型投票决策**：避免单一模型偏见
3. **质量优先**：不追求速度，追求质量
4. **输出必须重写**：原始数据必须经过模型重构
5. **知识图谱核心**：知识图谱贯穿全流程
6. **用户可配置**：灵活配置输出形式
